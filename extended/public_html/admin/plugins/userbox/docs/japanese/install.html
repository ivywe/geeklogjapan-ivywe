UserBox プラグイン

(2012/02/17update)

インストール

  1. データベースおよびファイルのバックアップをとります。
  2. 標準的なプラグインのファイル配置に従い、ファイルをアップします。
    1. plugins/　以下に　userbox以下をアップ
    2. admin/plugins/userbox/以下に　admin以下のファイルを移動
    3. public_html/userbox/以下に　public_html以下のファイルを移動
  3. Rootユーザーとしてログインし，プラグイン管理の画面でインストールを実行します

その他必要なフォルダ＆ファイル

  • public_html/images/userbox (追加項目の画像（外記憶）の記憶場所)
  • public_html/userbox_data (ファイル保存場所)
  • private/data/userbox/ (ファイル保存場所)

    • ファイル保存場所は、コンフィギュレーション設定で変更可能です

インストール時に作成される権限

  1. userbox.admin:UserBoxプラグインに関するすべての処理ができる。
  2. userbox.edit:(ユーザに更新を許可しない設定になっていても)更新ができる。
  3. userbox.joingroup:(ユーザにマイグループの更新を許可しない設定になっていても)更新ができる。
  4. userbox.user:UserBoxに登録するユーザ。ログインユーザを登録する設定になっている場合は、ログインユーザはすべて登録されます。

インストール時に作成されるグループ

  1. UserBox Admin :userbox.admin userbox.edit userbox.submit 権限を持つ
  2. UserBox Editor :userbox.edit　権限を持つ
  3. UserBox Manager :userbox.edit userbox.joingroup　userbox.user 権限を持つ
  4. UserBox User :userbox.user　権限を持つ

実行時に作成されるファイル

  • data/userboxconfig_bak.php　　コンフィギュレーションのバックアッップ

アンインストール方法

  • データベースおよびファイルのバックアップをとります。
  • Rootユーザーとしてログインし，プラグイン管理の画面で削除を実行します。
  • アップしたファイルを削除します。

アンインストールすると データベース上のUserBox専用データは、すべて削除されます。

通常のアップグレード方法

  1. データベースおよびファイルのバックアップをとります。
  2. 標準的なプラグインのファイル配置に従い、ファイルをアップします。
    1. plugins/　以下に　userbox以下をアップ
    2. admin/plugins/userbox/以下に　admin以下のファイルを移動
    3. public_html/userbox/以下に　public_html以下のファイルを移動
  3. Rootユーザーとしてログインし，プラグイン管理の画面で更新を実行します

基本的なテスト版のアップグレード方法

アンインストール、削除してから、あらためてインストールしてください。

アンインストールする前にコンフィギュレーション設定のバックアップを 作成しておくと、インストールしなおした際にリストアすることができます。

アンインストールすると データベース上の専用データは、すべて削除されます。

アンインストールすると アップロードしたファイルは、すべて削除されます。

テーブルの追加変更が必要な場合があります。

グループおよび権限の追加変更が必要な場合があります。
